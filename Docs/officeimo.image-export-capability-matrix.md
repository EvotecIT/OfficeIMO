# OfficeIMO Image Export Capability Matrix

This matrix tracks dependency-free image export across OfficeIMO document packages. It is intentionally implementation-facing: each item names the owning layer so improvements keep flowing into the shared rendering foundation instead of becoming separate Excel, PowerPoint, or Word renderers.

## Ownership Model

| Layer | Owns | Must not own |
| --- | --- | --- |
| `OfficeIMO.Drawing` | PNG/SVG rendering, image export result/options, raster canvas, dependency-free PNG, uncompressed top-down/bottom-up 24/32-bit BMP decode, first-frame GIF decode, BMP-to-PNG SVG data URI fallback, SVG formatting, image projection, frame transforms, text layout primitives, reusable border boxes, charts, nested drawing composition, diagnostics infrastructure | Excel workbook semantics, PowerPoint slide/master semantics, Word pagination semantics |
| `OfficeIMO.Excel` | Worksheet/range selection, cell style and value semantics, conditional formatting evaluation, anchors, page/print policy | Private PNG/SVG encoders or duplicate text/chart/image renderers |
| `OfficeIMO.PowerPoint` | Slide size, backgrounds, master/layout inheritance, shapes, text boxes, tables, chart snapshots, picture relationships | Duplicate chart renderer, duplicate transform engine, private image encoder |
| `OfficeIMO.Word` | Section/page policy, body/header/footer traversal, paragraph/table/image flow, first-page scope, Word-specific diagnostics | Duplicate drawing renderer, fake full pagination engine without Word layout semantics |

## Diagnostic Gaps To Burn Down

| Diagnostic | Current meaning | Preferred next move |
| --- | --- | --- |
| `unsupported-powerpoint-shape` | Shape type, geometry, transform bounds, text-margin case, or unsupported custom geometry command/formula case cannot be projected | Split by reason as fidelity grows; add Drawing geometry primitives where reusable |
| `unsupported-powerpoint-image-raster` | PNG output cannot decode source raster bytes beyond the shared PNG, uncompressed BMP, and first-frame GIF decoder set | Add dependency-free decoder support in Drawing if justified |
| `unsupported-powerpoint-image-svg` | SVG output cannot embed or transcode image bytes through the shared Drawing image policy | Keep in Drawing image policy |
| `unsupported-slide-background` | Background fallback used | Expand PowerPoint background projection, reuse Drawing fills/images |
| `unsupported-word-page-index` | Only first page preview is supported | Requires Word pagination/layout model |
| `unsupported-word-body-element` | Body element has no Drawing projection | Add Word-owned traversal plus Drawing primitive mapping |
| `unsupported-word-nested-table` | Nested table shape is outside the direct/simple cell-local layout currently projected | Expand cell-local flow/table layout deliberately |
| `unsupported-word-shape` | Shape geometry, size, or wrap family is not currently projected through shared Drawing | Expand Word-owned shape extraction and route reusable geometry/style into Drawing |
| `unsupported-word-textbox` | Text box size, anchor, or frame cannot be projected through shared Drawing | Expand Word-owned text-box layout/effects while continuing to use Drawing text/shape primitives |
| `unsupported-word-floating-image` | Unresolved anchor or page-outside anchor requires Word anchoring/text-wrap layout beyond the current first-page image slices | Add a Word-owned floating layout model, then project resolved placement through Drawing images |
| `limited-word-floating-image-wrap` | Tight/through wrapped image rendered with a rectangular exclusion instead of true polygon or transparent-region wrapping | Add Word-owned polygon/transparent-region wrapping and remove this fallback warning when fidelity is real |
| `unsupported-word-image-raster` | PNG output cannot decode source raster bytes beyond the shared PNG, uncompressed BMP, and first-frame GIF decoder set | Add decoder support in Drawing |
| `unsupported-word-image-svg` | SVG output cannot embed or transcode image bytes through the shared Drawing image policy | Keep in Drawing image policy |
| `unsupported-word-pagination` | Content overflowed first-page flow | Replace with real pagination once Word layout model exists |

## Near-Term Fidelity Order

1. Keep Excel as the parity benchmark and avoid regressing range export behavior.
2. Push PowerPoint next because fixed-size slides map naturally to Drawing.
3. Improve shared Drawing geometry and chart fidelity whenever PowerPoint/Excel need the same concept. Common and expanded DrawingML polygon/star, flowchart, callout, arrow-callout, symbol, bracket/brace, multi-direction arrow, and additional arrow presets now live in `OfficeShapePresets`, keeping fixed-slide diagram support reusable.
4. Build Word layout in deliberate layers: first richer paragraph/run/list/table-cell layout, then floating objects, then real pagination. Basic body, header/footer, and table-cell list markers now ride the existing `DocumentTraversal` numbering model and Drawing text primitives; consecutive rich text runs, tab expansion, paragraph left/right/first-line/hanging indentation, direct/style-inherited/document-default before-after paragraph spacing, direct/style-inherited contextual spacing collapse, plain/rich-text justification, direct theme paragraph/rich/table-cell text, direct run highlights, direct paragraph shading/borders, opaque Wordprocessing hex-backed paragraph colors, table cell text padding, multi-paragraph table-cell spacing/contextual collapse, direct/simple nested tables, independent/double/diagonal table-cell borders, direct theme table fills/borders, conditional table band sizes, table corner precedence, first-page header/footer tables, inline PNG images, square-wrapped PNG anchors, square-wrapped DrawingML shape anchors, and square-wrapped text boxes, inline and square-wrapped anchored DrawingML preset shapes, basic VML primitive shapes, DrawingML/VML shape transforms, DrawingML/VML text-box frame and text projection with VML inset, fit-to-text height, vertical anchor, rich-run, and transform evidence, Word image crop, page-relative top-and-bottom anchored PNG images, page-relative no-wrap anchored PNG images, square-wrap anchored PNG images with authored left/right side preference and anchor distance margins, and tight/through anchored PNG images with rectangular exclusion fallback now use shared Drawing primitives. Real tight/through polygon or transparent-region wrapping remains diagnosed until Word owns that layout. PowerPoint basic bullet and number markers, including alpha and roman auto-number schemes plus level-derived hanging indents, direct text-box paragraph spacing/line spacing, direct table-cell paragraph spacing/line spacing and list markers through the friendly `PowerPointTableCell` paragraph/list API, straight, common bent, curved, and native connection-shape connectors, plain/rich-text justification, scheme-colored auto-shape and text-box frame fills/outlines, scheme-colored table fills, scheme-colored table borders, merged-cell spans, table-style whole-table/first-row/first-column/last-column/corner/banded-row/banded-column fills and span-aware style borders, diagonal table borders, scheme-colored text, and direct/scheme-colored run highlights now project through the same shared shape/text/rich-text/border renderer paths while paragraph and theme semantics stay package-owned.
5. Expand fluent APIs only as thin adapters over proven export paths. Common convenience profiles, export aliases, byte aliases, and save aliases now live on the Drawing builders so all document packages inherit the same ergonomics.
